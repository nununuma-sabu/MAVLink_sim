# 3D描画のモダンOpenGL化方針

`MapView3D` は現在、Qt の `QOpenGLWidget` 上で固定機能パイプライン
（`glBegin` / `glEnd`、`glMatrixMode`、`glColor` など）を使って描画しています。
実装は直感的ですが、OSM由来の建物・道路・線路が増えた場合にCPU負荷とドライバ呼び出し回数が増えやすいため、段階的に VBO とシェーダーベースの描画へ移行します。

## 目的

- 都市部の建物数増加に耐えられる描画性能にする
- 建物、道路、線路、地面をバッチ描画し、OpenGL呼び出し回数を削減する
- 将来的なマテリアル、ライト、ランドマーク表現の追加をしやすくする
- Qt5環境でも動く範囲で、固定機能パイプラインへの依存を減らす

## 移行方針

一度に全描画を置き換えず、動的要素と静的要素を分けて進めます。

### Phase 1: 静的都市メッシュの分離

- 完了: `Map3DMeshBuilder` / `Map3DStaticMesh` を追加し、建物、道路、線路、地面の頂点生成を `MapView3D` から分離済み
- 建物、屋根、道路、線路、地面を `MapView3D` から切り出す
- OSMデータ読み込み完了時に頂点配列を生成する
- 頂点には位置、色、必要に応じて法線を持たせる
- ドローン、HUD、ウェイポイントなどの動的描画は既存実装のまま残す

### Phase 2: VBO導入

- 完了: `Map3DStaticMesh` をGPU上のVBOへアップロードし、静的都市要素を `glDrawArrays` で描画するように移行済み
- 静的都市メッシュをOpenGLのVBOにアップロードする
- `glDrawArrays` / `glDrawElements` で描画する
- 建物壁面、屋根、道路帯、線路、地面をプリミティブ種別ごとにバッチ化する
- 地点切り替え、OSM再取得、キャッシュ読み込み時だけVBOを再構築する

### Phase 3: シェーダー導入

- `QOpenGLShaderProgram` で基本シェーダーを追加する
- `modelViewProjection`、頂点色、フォグ係数をシェーダー側へ移す
- 固定機能の `glMatrixMode`、`glColor`、`glFog` 依存を段階的に撤去する
- 既存のQPainter HUDは2Dオーバーレイとして維持する

### Phase 4: 動的要素の移行

- ドローンモデル、飛行トレース、ウェイポイントを小さな動的VBOへ移す
- トレースはリングバッファ化して更新コストを抑える
- プロペラなどアニメーション要素はインスタンス化または小メッシュ更新で扱う

## 優先してVBO化する対象

1. 建物壁面・屋根
2. 道路帯・道路マーキング
3. 線路・枕木
4. 地面テクスチャ風ポリゴン
5. 飛行トレース・ウェイポイント
6. ドローンモデル

## 完了条件

- 都市部プリセットで描画がカクつきにくくなる
- OSM建物数が多い地点でも、静的メッシュの描画呼び出しが少数にまとまる
- `MapView3D` から都市メッシュ生成と描画責務が分離されている
- 新しい建物ディテールやランドマーク描画を追加しやすい構造になっている

## 注意点

- Qt5 / OpenGL互換プロファイルで動く環境を維持する
- 既存の見た目を維持しながら小さく置き換える
- まずは静的都市メッシュだけを対象にし、ドローンやHUDは後回しにする
